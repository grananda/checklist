El change ui-mvp-checklist entrega la pantalla única usable del MVP (HU-01..HU-06) consumiendo la API REST de tareas: ver lista, añadir, marcar/desmarcar, editar, borrar con confirmación y ver el progreso, con accesibilidad WCAG 2.1 AA. Cierra la Fase 1.

#Supuestos y huecos

  • Supuesto: el actor primario es la Persona del equipo que opera la web; el Servidor API (tareas-api) es un participante de sistema ya existente.
  • Supuesto: la sincronización aplica al store la respuesta del servidor (sin optimistic ni tiempo real en esta fase).
  • Hueco: reordenar (HU-07) y reiniciar (HU-08) NO entran (Fase 4); la propagación en tiempo real entre clientes (HU-09) tampoco (Fase 5). Dos pestañas se reconcilian por recarga.
  • Hueco: el progreso es estado derivado en el cliente; no hay endpoint de progreso.

#Casos de uso

La persona del equipo opera el CRUD del MVP desde la pantalla única. include marca la confirmación previa del borrado y la actualización del progreso que comparten las mutaciones.

#Actividad

Recorrido típico de la persona en la pantalla, desde la carga hasta operar la lista. La validación del formulario da feedback antes de llamar a la API.

#Secuencia 1 - Happy path: añadir una tarea

La persona añade una tarea desde el modal; el store llama a la API y aplica la tarea devuelta. Precondición: la web está cargada.

#Secuencia 2 - Borrar con confirmación

El borrado exige confirmación previa; solo al confirmar se llama a la API. Precondición: la tarea existe en la lista.

#Secuencia 3 - Marcar y progreso (aria-live)

Marcar una tarea actualiza el estado y el indicador de progreso, anunciado por aria-live.

#Estado

Estado de la pantalla según el contenido de la lista y los modales abiertos.
