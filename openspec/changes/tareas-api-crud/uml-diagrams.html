Change tareas-api-crud · Generado el 2026-06-24

El change tareas-api-crud implementa el backend del MVP (HU-01..HU-06): la API REST y el dominio de las tareas de la única lista compartida (listar, crear, marcar/desmarcar, editar, borrar) con validación de límites y cálculo de progreso, persistido en SQLite. La validación de forma vive en el transporte (JSON Schema) y la semántica en el servicio.

#Supuestos y huecos

  • Supuesto: el actor primario es el Cliente API (la UI del MVP de la Fase 3 o cualquier consumidor REST). En este change no hay interfaz de usuario; las pantallas llegan en ui-mvp-checklist.
  • Supuesto: el cálculo de progreso (HU-06) es estado derivado que el cliente obtiene de la lista; el servidor expone el dominio pero no un endpoint de progreso en esta fase.
  • Hueco: la difusión en tiempo real de las mutaciones (HU-09) NO forma parte de este change; tras persistir solo se responde. Se aborda en la Fase 5 (tiempo-real).
  • Hueco: reordenar (HU-07) y reiniciar (HU-08) quedan fuera de alcance (Fase 4).

#Casos de uso

El Cliente API opera el CRUD del MVP sobre la lista compartida. Las relaciones include marcan la validación obligatoria (forma en transporte, semántica en servicio) y la persistencia que comparten las mutaciones.

#Actividad

Flujo de una mutación POST (crear tarea): la ruta valida la forma; el servicio valida título y límites (NFR-12), genera id y posición al final, y el repositorio persiste. Errores de dominio devuelven 400 en español.

#Secuencia 1 - Happy path: crear tarea

Alta de una tarea válida. Precondición: la app Fastify está escuchando y la tabla tareas existe.

#Secuencia 2 - PATCH parcial (marcar o editar)

PATCH /api/tareas/:id aplica solo los campos presentes (hecha, titulo, descripcion). Campo ausente no cambia; descripcion vacia la vacia. Precondición: la tarea existe.

#Secuencia 3 - Validación: título vacío

Una creación con título vacío o solo espacios pasa la forma (es string) pero la rechaza el servicio con mensaje en español.

#Estado

Ciclo de vida de una Tarea. Nace pendiente; marcar/desmarcar alterna estado; editar no cambia estado ni posición; borrar la elimina de forma definitiva.
